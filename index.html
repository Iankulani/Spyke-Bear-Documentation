Installation · GitHub

Deploy Spyke Bear in minutes. The bot listens for authenticated commands and executes security actions (network scans, log checks, threat intel, etc).

1. Clone from GitHub

git clone https://github.com/Iankulani/spykebear.git
cd spykebear

2. Environment setup

# Copy example env
cp .env.example .env
# Edit with your API keys for Telegram, Slack, Discord, iMessage bridge
nano .env

3. Docker / Node

# Using python(18+)
python spykebear.py


# or Docker (recommended)
docker-compose up -d

4. Connect platforms

  • Telegram: Create bot via @BotFather, set TELEGRAM_TOKEN
  • Slack: App + Socket Mode, SLACK_BOT_TOKEN, signing secret
  • Discord: Bot token + Privileged Gateway Intents
  • iMessage: Use AppleScript bridge / Mac mini relay (iMessage extension)
  • Web: Built-in dashboard on port 3000 with live command UI

After setup, Spyke Bear auto-connects to all channels. Every command is encrypted & audit logged.

Fire Commands · Syntax

Send commands from any integrated chat or web terminal. Spyke Bear responds in real-time with cyber intelligence and system actions.

🔹 Core Command Structure

/spyke <action> [target] [options]

Examples:
/scan 192.168.1.0
/nmap 127.0.0.7
/analyze 127.0.0.7
/curl 
/spyke status

Platform-specific usage

Telegram/Slack/Discord: Use /spyke prefix or mention @SpykeBear.
iMessage: Text spyke: command to configured number.
Web Terminal: Access https://your-spyke.domain/terminal with JWT token.

🔐 All traffic protected with TLS + end-to-end verification

Step-by-Step Web + Next.js Deployment

npx create-next-app@latest spyke-web
cd spyke-web
npm install socket.io-client axios

Integrate API routes that forward commands to Spyke core.

🔌 2. Backend bridge

// pages/api/command.js
import { executeSpykeCmd } from '../../lib/bearCore';
export default async function handler(req,res) {
  const {cmd} = req.body;
  const output = await executeSpykeCmd(cmd);
  res.status(200).json({result: output});
}
